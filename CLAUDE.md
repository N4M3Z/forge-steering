# CLAUDE.md

> `CLAUDE.md` and `AGENTS.md` are autogenerated by `/Init`. Do not edit directly -- run `/Update` to regenerate.

AI Steering Rules in Statement/Bad/Correct format. Provides behavioural guardrails derived from real failures — tells AI tools how to approach tasks, what mistakes to avoid, and what correct behaviour looks like.

## Skills

| Skill | What it does |
|-------|-------------|
| **BehavioralSteering** | AI guardrails — simplicity, scope discipline, opponent review, formatting conventions |

## Install / Verify

```bash
make install          # deploy skills (Claude, Gemini, Codex, OpenCode)
make verify           # check skills deployed across all providers
make clean            # remove previously installed skills
make test             # validate-module convention checks
```

## Architecture

```
skills/
    BehavioralSteering/   SKILL.md + SKILL.yaml
lib/                      git submodule -> forge-lib (Rust binaries for deployment + validation)
defaults.yaml             skill roster
config.yaml               user overrides (gitignored), same structure as defaults
module.yaml               module metadata (name, version)
.claude-plugin/           plugin.json manifest for Claude Code plugin discovery
```

## Configuration

`config.yaml` (gitignored) points at vault steering directories:

```yaml
steering:
    - "Orchestration/Behaviour"
```

When `steering:` paths contain subdirectories with `SKILL.md` files, `forge-update.sh` auto-injects them into `plugin.json`. With no paths configured, only the built-in `BehavioralSteering` skill loads.

## Conventions

### Rule Format

Each rule follows the Statement/Bad/Correct pattern adopted from [PAI](https://github.com/danielmiessler/Personal_AI_Infrastructure)'s AI Steering Rules:

- **Statement** — what should happen
- **Bad** — a concrete example of wrong behaviour
- **Correct** — a concrete example of right behaviour

### Git

Conventional Commits: `type: description`. Lowercase, no trailing period, no scope.

Types: `feat`, `fix`, `docs`
